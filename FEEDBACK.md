# 프로젝트 현황 분석 및 향후 개발 피드백 (FEEDBACK.md)

## 1. 개요
현재 `PLAN.md`에 정의된 메인 페이지의 레이아웃과 핵심 컴포넌트(Hero, Job Cards) 구현은 완료되었습니다. 그러나 헤더와 푸터에 연결된 링크들이 실제 페이지로 구현되지 않아 **404 Not Found** 오류가 발생하는 상태입니다. 이를 해결하고 완성도 높은 클론 사이트를 만들기 위한 상세 피드백입니다.

## 2. 미구현 페이지 (Missing Pages & 404s)
현재 UI에는 링크가 존재하지만, 실제 라우팅 파일이 없어 접근 불가능한 페이지들입니다.

### 2.1. 채용 정보 카테고리 페이지
헤더 네비게이션에 있는 메뉴들입니다. 메인 페이지와 유사하지만, 특정 카테고리(FC, TMR 등)로 필터링된 리스트를 보여주는 페이지가 필요합니다.
- [ ] `/fc` (FC 모집 리스트 페이지)
- [ ] `/tmr` (TMR 모집 리스트 페이지)
- [ ] `/general` (총무/일반직 모집 리스트 페이지)
- [ ] `/jobs/[id]` (채용 공고 상세 페이지 - **핵심 기능**)

### 2.2. 인재 정보 및 회원 서비스
- [ ] `/talents` (인재정보 리스트 페이지)
- [ ] `/login` (로그인 페이지)
- [ ] `/signup` (회원가입 페이지)

### 2.3. 푸터 정보 페이지
사이트 신뢰도를 위한 법적 고지 및 안내 페이지입니다.
- [ ] `/terms` (이용약관)
- [ ] `/privacy` (개인정보처리방침)
- [ ] `/ads` (배너광고/이용문의 안내)

### 2.4. 시스템 페이지
- [ ] `not-found.tsx` (커스텀 404 에러 페이지 - 현재 Next.js 기본 404가 뜸)

## 3. 기능적 미비점 (Functional Gaps)

### 3.1. 검색 기능 로직 부재
- **현상:** Hero 섹션의 검색창(시/도 선택, 직종 선택)이 UI만 있고 작동하지 않음.
- **필요:** 검색 버튼 클릭 시 `/search?region=...&type=...` 형태로 이동하거나, 리스트 페이지에서 필터링된 결과를 보여주는 로직 필요.

### 3.2. 상세 페이지 연결 부재
- **현상:** 채용 공고 카드(`PremiumJobCard` 등)를 클릭해도 아무런 반응이 없거나 이동하지 않음.
- **필요:** 카드 클릭 시 `/jobs/${job.id}`로 이동하는 링크(`Link` 컴포넌트) 적용 및 상세 페이지 구현 필요.

### 3.3. 반응형 모바일 메뉴 인터랙션
- **현상:** 모바일 뷰에서 상단 메뉴가 가로 스크롤로만 되어 있음.
- **제안:** 로그인/회원가입 등은 햄버거 메뉴(Drawer)로 숨기거나, 모바일 전용 하단 네비게이션 바(Bottom Tab Bar) 고려.

## 4. 향후 개발 우선순위 제안

1.  **공통 레이아웃 페이지 생성 (`/fc`, `/tmr` 등):** 재사용 가능한 `JobLayout` 컴포넌트를 만들어 각 카테고리 페이지 빠르게 구축.
2.  **공고 상세 페이지 (`/jobs/[id]`):** 지원하기 버튼, 상세 요강, 담당자 연락처가 포함된 상세 뷰 구현.
3.  **로그인/회원가입 UI:** 기능은 Mock으로 처리하더라도 폼(Form) UI 구현.
4.  **커스텀 404 페이지:** 사이트 브랜드(Insumatch)에 맞는 친절한 에러 페이지 구현.

---
**결론:** 현재 메인 페이지는 완성도가 높으나, 사이트로서 기능하기 위해서는 **서브 페이지(Sub-pages)**의 구축이 시급합니다. 다음 단계로 **카테고리별 리스트 페이지**와 **상세 페이지** 구현을 추천합니다.
